# 可借鉴到自研后端的技术清单

## 1. 使用方式说明
本清单按“可借鉴技术 -> 在你项目的落地方式 -> 成本/风险”组织，重点面向你这种“需要技能执行 + 沙箱 + API 搜索 + 文件处理 + 数据分析”的后端架构。

## 2. 核心清单（优先级从高到低）

|---|---|---|---|---|

## 3. 对你当前 E2B 资源最有价值的借鉴

### 3.1 立刻可做
- 统一 Skill Runner：skill 触发后全部在 E2B 会话执行。
- 每次执行固定输出 `run_id`、`tool_calls`、`artifacts`、`cost`。
- 将高风险工具（命令执行、文件删除、外网发送）挂审批策略。

### 3.2 第二阶段
- 引入 marketplace 安装治理（扫描、签名、审计、回滚）。
- 引入 session_search 供跨会话业务追问。
- 引入记忆 provider（DB）支撑 C 端长期个性化。

## 4. 不建议直接照搬的部分
- 单文件超大编排文件（迁移时应拆服务模块）。
- 本地文件型 memory 作为长期最终形态（生产建议数据库化）。
- 将训练评测链（Atropos）直接耦合进线上请求路径。

## 5. 你可以直接复用的抽象接口（建议）

```text
ToolRegistry
  - register(schema, handler, policy)
  - resolve(tool_name)
  - dispatch(tool_call, context)

SkillRuntime
  - listSkills(filter)
  - loadSkill(skill_id)
  - executeSkill(skill_id, input, sandbox_ctx)

MemoryProvider
  - prefetch(query, session_id)
  - syncTurn(user, assistant, session_id)
  - searchSessions(query, user_id)
```
