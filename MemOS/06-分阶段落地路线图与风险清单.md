## TL;DR

目标应是：

- 先把 MemOS 变成稳定记忆底座
- 再补 skill 执行引擎
- 最后再做 marketplace 运营化

## 1. 分阶段路线图（含退出标准）

| 阶段 | 时间 | 核心目标 | 关键交付 | 退出标准 |
|---|---|---|---|---|
| Phase 0 | 1-2 周 | 基线稳定 | 架构梳理、监控、备份 | 核心接口稳定运行 7 天 |
| Phase 1 | 2-4 周 | 长期记忆升级 | chunk/embedding/hybrid search | Recall@10 和 P95 达标 |
| Phase 2 | 3-6 周 | skill runtime | sandbox + policy + audit | 高风险 skill 可控执行 |
| Phase 3 | 4-8 周 | marketplace MVP | 发布/审核/安装/版本 | 可安全上架并灰度发布 |
| Phase 4 | 持续 | 企业化治理 | 合规、成本、质量闭环 | 有稳定运营仪表盘 |

## 2. 每阶段必做事项

### Phase 0：打基础

1. 标准化日志和 trace-id。
2. 补关键链路告警（DB、队列、MCP error）。
3. 完成恢复演练（数据库备份恢复）。

### Phase 1：做记忆能力

1. 建 memory 相关新表。
2. 引入异步 embedding worker。
3. 上线 hybrid recall API。
4. 增加删除一致性任务。

### Phase 2：做 skill 执行能力

1. 建 skill registry 和执行契约。
2. 上线代码执行沙箱。
3. 外部 API skill 上密钥托管。
4. 建立调用审计和回放。

### Phase 3：做 marketplace

1. skill 包签名 + 扫描。
2. 审核流和灰度发布。
3. 版本兼容矩阵。
4. 评分与下线机制。

## 3. 风险清单（按严重度）

| 风险 | 严重度 | 触发信号 | 预防措施 |
|---|---|---|---|
| 代码执行逃逸 | 高 | 异常系统调用/网络外联 | 容器隔离 + seccomp + 网络白名单 |
| 记忆污染 | 高 | 错误事实反复命中 | 事实置信度 + 人工纠错 + 回滚 |
| 成本失控 | 高 | embedding/搜索账单暴涨 | 配额、缓存、批处理、分层索引 |
| 数据合规问题 | 高 | 删除请求无法彻底生效 | 端到端删除链路 + 审计 |
| 召回质量差 | 中 | 用户反馈命中不准 | Hybrid + rerank + 指标闭环 |

## 4. 前 10 个任务（可直接进 Jira）

1. 新建 `memory_chunk` 表和迁移脚本。
2. 新建 `memory_embedding` 表并接 pgvector。
3. 增加 memo 更新后的异步事件。
4. 开发 embedding worker（支持重试）。
5. 上线 `memory/search`（hybrid）。
6. 增加 skill manifest 校验器。
7. 实现 code skill 沙箱执行器。
8. 增加 skill 调用审计表。
9. 接入 MCP tool 调用指标。
10. 增加“用户记忆删除”全链路任务。

## 5. 验收 KPI

1. Recall@10（记忆召回命中率）
2. Skill 执行成功率
3. P95 响应时延
4. 每用户日均成本
5. 数据删除成功率

一句话：先把“能用”做到极致，再做“可扩展”和“可运营”。
